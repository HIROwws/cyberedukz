# Day 1 Kickoff

## Objective

Prepare the project foundation so the next task can start with implementation instead of planning.

## Today Must Be Done

- Fix MVP scope.
- Create Jira project structure.
- Import or manually create backlog.
- Choose stack and repository layout.
- Draft database schema.
- Define the first demo user journey.

## MVP Scope

### Included

- Registration and login
- User roles: student and admin
- Course catalog
- Course detail page
- Lesson page
- Challenge page with flag submission
- Progress and points
- Leaderboard
- Basic admin content management
- Russian UI text
- Kazakhstan/CIS context in examples

### Not Included

- Real virtual machines
- Paid subscriptions
- Certificates
- Multi-tenant university groups
- Complex LMS integration
- Production-grade lab isolation

## First Demo User Journey

1. User creates an account.
2. User opens the course catalog.
3. User starts `Web Security Basics`.
4. User reads the first lesson.
5. User submits a flag.
6. System awards points and updates progress.
7. User sees their place in the leaderboard.

## Day 1 Definition of Done

- Jira board exists.
- Backlog is imported from `deliverables/jira_backlog_cyberedukz.csv` or created manually.
- Repository contains README, architecture notes, environment template, and database schema.
- Day 2 task is clear: implement registration/login and protected user session.
